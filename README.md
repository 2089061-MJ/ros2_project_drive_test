# ROS2 주행 프로젝트 알고리즘 시행착오
시행착오 부분을 정리한 repo입니다.


### A*의 한계
A* 알고리즘은 정적 환경을 가정하고 시작 지점부터 목표 지점까지의 최단 경로를 탐색하는 방식이므로, 경로 중간에 새로운 장애물이 발생할 경우 현재 위치에서 목표 지점까지의 전체 경로를 다시 계산해야 합니다. 이로 인해 동적 장애물에 대한 실시간 대응이 어렵고, 실제 주행 환경에서는 경로 재계산 지연으로 인해 장애물을 회피하지 못하는 문제가 발생하여 선택하지 못하였습니다. 

### DWA의 한계
좁은 통로를 지나거나 목표 지점에 가까워질 때, 로봇이 좌우로 심하게 흔들리는 현상이 자주 발생하였습니다.
또한 매 순간 최적의 경로가 미세하게 바뀌기 때문에 로봇이 결정을 못 내리고 갈팡질팡하는 모습을 보였습니다.

### TEB의 한계
라즈베리파이에 적용해보았으나 높은 연산량으로 로봇의 주행이 delay되는 문제가 발생(주행중 벽에 그대로 충돌)
